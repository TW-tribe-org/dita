Restricting tasks to one procedure only

Task topics can only contain one procedure, with one continuous set of numbered steps.

The task information type is specifically designed to contain one procedure only. If you feel you need to document two procedures in one task topic, you should re-examine your approach. Two procedures should be documented in two task topics.
If necessary, you can output two DITA task topics to one single output HTML topic, using the chunk attribute.
If you feel that you need to restart numbering within a task, then you probably have more than one procedure in the topic, and should split them into two separate task topics.
In some cases, what appears to be two procedures may turn out to be a nested procedure, in which case the substeps element can be used. The choicetable structure is also useful for procedures where a step has a number of alternative options.
If a procedure can be performed in two different ways, you should document those alternatives in two separate topics.

DITA tasks are deliberately very constrained, partly to disallow what it considers to be lazy topic-writing. (Not that your example is lazy, but it has the same structure as lazy tasks, and the DITA DTD can't tell the difference. You are just collateral damage.) So, the short answer is "you can't do that". It's hard to say what is the best way of doing what you want. There have been past discussions about this kind of thing on dita-users. What works is going to depend on the factors that you stripped out of your example to get your point across, so you may just have to try different ways of writing the task until you get something that feels right. If I were the author (and if I were actually writing about opening windows, rather than the real task that you have in mind), I'd probably create two tasks ("Opening the window with the hand" and "Opening the window with the foot"), and wrap them in a container task that itself has no <steps> element. Or I would decide that this isn't really a task, and use standard <ol> and <li> in a concept. Or I would see if I could get away with just using a <note> in step 1 ("You can also use your leg"). Or I would use <choices> or <choicetable> in step 1 to list the body parts that are suitable for window-opening.

Deborah Pickett, Yahoo! DITA Users Group

Personally, I would lean towards using a choicetable with headings like, Limb and Method. In the left column would be Arm and Leg and in the right column would be ordered lists with the steps. This makes the entire task boil down to a single unordered step with the command Open the window followed by the choicetable.
